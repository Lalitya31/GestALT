╔═══════════════════════════════════════════════════════════════════════════════╗
║                         EVALUATION FLOW                                        ║
╚═══════════════════════════════════════════════════════════════════════════════╝

1. User modifies component property (e.g., button width)
   ↓
2. ChallengeRenderer.onComponentUpdate() triggered
   ↓
3. LearningSystem.recordDecision() - Track the change
   ↓
4. UI re-renders with new component state
   ↓
5. User clicks "Evaluate Design"
   ↓
6. PerceptionEngine.analyze(components) calculates:
   ├─ Attention map (who sees what first)
   ├─ Hierarchy strength (clarity of importance)
   ├─ Cognitive load (mental effort)
   ├─ Accessibility metrics (WCAG compliance)
   └─ Returns PerceptionMetric object
   ↓
7. Challenge.evaluate(components, engine) processes:
   ├─ Get metrics from engine
   ├─ Calculate domain-specific score
   ├─ Check success criteria (e.g., hierarchyStrength > 0.6)
   ├─ Generate explanations (which rules fired and why)
   └─ Returns Outcome object
   ↓
8. GestALTApp.displayMetrics() shows:
   ├─ Metric values with visual indicators
   ├─ Explainability cards (issue, reason, suggestion)
   └─ Accessibility report (if applicable)
   ↓
9. If challenge passed:
   ├─ LearningSystem.completeChallenge()
   ├─ Update user skill profile (+5 to +10 points)
   ├─ Save to localStorage
   └─ Select next adaptive challenge

╔═══════════════════════════════════════════════════════════════════════════════╗
║                    KEY ENGINEERING DIFFERENTIATORS                             ║
╚═══════════════════════════════════════════════════════════════════════════════╝

1. PERCEPTION AS SIMULATION
   Not "does this look good?" but "where does attention flow and why?"
   
2. DETERMINISTIC EVALUATION
   Same input always produces same output - traceable, explainable
   
3. ADAPTIVE INTELLIGENCE
   System learns user weaknesses and adapts challenge selection
   
4. ACCESSIBILITY AS CONSTRAINT
   Real-time validation, not optional checklist
   
5. EXPLAINABILITY FIRST
   Every metric shows calculation, every failure shows reason
   
6. CLEAN DOMAIN MODELING
   Explicit entities with clear responsibilities and relationships
   
7. STANDARDS COMPLIANCE
   Implements actual WCAG 2.1 formulas, not approximations
